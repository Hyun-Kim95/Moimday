# ADR-0001: MVP 제품 결정 (D-1 ~ D-8)

**상태:** Accepted  
**날짜:** 2026-05-20  
**근거:** [moimday-prd.md](../requirements/moimday-prd.md) §3.3

## 컨텍스트

Moimday MVP 범위·정책을 구현 전에 고정해야 Gate 1·2 및 병렬 개발이 가능하다.

## 결정

| ID | 결정 |
|----|------|
| D-1 | 가족 그룹 최대 **10명** (→ **30명·다중 그룹**은 [0003](0003-multi-group-30-members.md)) |
| D-2 | 인증: **전화번호 OTP** |
| D-3 | MVP에 **날짜·시간 후보 투표** 포함 |
| D-4 | 리마인더 **다회**(미응답자, 단계별 상한, §8.1) |
| D-5 | 커뮤니케이션: **모임 댓글만** |
| D-6 | **14세 미만** 구성원 없음(보호자 플로우 비포함) |
| D-7 | **한국어** UI·문서만 |
| D-8 | 일시 **조율자(모임 생성자) 수동 확정**; 자동 확정 없음 |

## 결과

- PRD v0.2~v0.5 및 `docs/api`, `docs/design`, `docs/qa` 초안이 본 결정을 따른다.
- 변경 시 PRD 개정 + 본 ADR Superseded 처리.

## 대안 (기각)

- 카카오 로그인 only → 가족 전화번호 통일 목적에 OTP 유지
- 자동 최다 득표 확정 → D-8로 기각
